# Format Competitor Style References

The `video-editor` skill reads this file to apply our format competitors' edit styles to a 100x Marketer video. Update this file whenever we observe new patterns from the listed creators, or when we add a new reference creator.

When in doubt, the spine of every 100x edit is **demo-first, screen-recording-heavy, on-screen text every 4-8 seconds, no fade-in, no intro card, single CTA card at the end**.

---

## @kylewhitrow (Vibe Marketing | Claude Code)

**Profile:** [instagram.com/kylewhitrow](https://www.instagram.com/kylewhitrow/). Bio: "Claude Code | Vibe Marketing." YouTube reference: "My Claude Code System For Viral Instagram Reels."

**Why we study him:** Closest format-and-niche match in the space. Already proving the audience exists for Claude-Code-for-marketers short-form.

### Visual style

- **Dominant visual:** screen recording of Claude Code (dark terminal theme) + an editor (VS Code or similar). Talking head is secondary, often picture-in-picture or A-roll cutaways.
- **Cut cadence:** ~1.5-2.5 seconds per shot on screen recording. Faster than the average Reel.
- **On-screen text:** large bold sans, white with subtle drop shadow, positioned in upper-third or center. Text appears with a snap-cut (no soft fade-in).
- **Color/look:** neutral, slight desaturation. Terminal blacks are deep. No film grain or trendy LUTs.
- **A-roll framing:** medium close-up, casual handheld feel. Not a tripod-locked vlog frame.

### Hook style

- Cold open on a result or claim. First 3 seconds is screen content + on-screen text overlay, not a face.
- Hook patterns we observe: "I built [thing] in [time]", "Here's how I [outcome]", "Stop using [tool], use [Claude Code thing] instead".

### Pacing

- 30-50 second videos most common.
- Visible cut every 2-3 seconds, no exceptions.
- Holds on prompts/configs for 2.5-4 seconds (the pause-and-copy moment).

### CTA pattern

- Verbal in the last 4-5 seconds + on-screen handle/community link.
- Cleanly cut, not faded.

### What to apply to 100x edits

- Default to screen-recording-as-primary-visual.
- Cold open with the result, not a face.
- Use Claude Code's dark theme as the natural visual identity. Do not theme over it; the terminal IS the brand.
- 1.5-3 second cut cadence on screen recordings.

## @mavgpt (Maverick Maltin)

**Profile:** [instagram.com/mavgpt](https://www.instagram.com/mavgpt/). 580K followers. Bio framing: AI & ChatGPT, daily prompts/tips/tools.

**Why we study him:** Adjacent niche (ChatGPT, not Claude Code), but he runs the dominant short-form AI teaching format at scale. The "pause and copy this prompt" beat is his signature.

### Visual style

- **Dominant visual:** full-screen prompt or chat output on a dark background. The prompt IS the video.
- **Cut cadence:** slower than kylewhitrow when the prompt is on screen (3-5 second holds). Faster on connector beats.
- **On-screen text:** very large, bold, often the prompt itself rendered as the on-screen text. Bottom-third has captions; top has prompt.
- **Color/look:** dark mode everything. High contrast white-on-black for prompts.
- **A-roll:** minimal. Often no face at all; voiceover only. When face appears, it's a small inset.

### Hook style

- The prompt itself is the hook. First 1-2 seconds is the prompt rendered full-screen, voiceover starts immediately: "Pause. Copy this."
- Hook patterns: "Pause and copy this prompt", "This [model] prompt does [outcome]", "Try this in [tool]".

### Pacing

- 25-45 seconds typical.
- The prompt holds for 3-5 seconds (the entire point of the format).
- After the prompt: 1-2 connector beats showing what it produces, then CTA.

### CTA pattern

- "Follow for daily prompts" verbal + handle on screen.
- Daily cadence is the differentiator. He is teaching the audience he ships every day.

### What to apply to 100x edits

- Use this format for any brief whose teaching beat is a Playbook (a prompt or skill the viewer can copy).
- Render prompts as full-screen text overlays, not screen recordings of typing them. Bigger, more readable.
- Holds of 3+ seconds on prompts are required, not optional.
- Voiceover-first edits are valid and often stronger than face-on-camera for prompt-teaching content.

## Hybrid edits (most 100x videos)

Most 100x videos blend the two. A typical hybrid:

- **0-3s:** mavgpt-style hook (full-screen claim or result, no face)
- **3-15s:** kylewhitrow-style screen recording showing the build/run
- **15-25s:** mavgpt-style prompt/output reveal (hold 3s for pause-and-copy)
- **25-40s:** kylewhitrow-style screen recording showing the result
- **40-50s:** A-roll talking head close (Brendan's face, 5s) + CTA card (5s)

Use this hybrid as the default skeleton when the brief doesn't dictate a clear style.

## Things we explicitly do NOT borrow

- **Fade-ins, zoom intros, logo splashes.** Cold open every time.
- **Talking-head-only intros.** Show the work, not the face, in the first 3 seconds.
- **Generic "viral hook compilation" cadence.** Every cut should serve the teach, not just keep the eye moving.
- **Dramatic music swells.** Music is a quiet bed under VO, never the lead.
- **Trendy filter LUTs / heavy color grades.** Neutral. The terminal and the artifact are the aesthetic.
